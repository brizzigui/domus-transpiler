Grammar (GLC) for .domus language

Program -> Automations
Automations -> Automation Automations | epsilon

Automation -> 'create' STRING ':' Fields
Fields -> Field Fields | epsilon

Field -> 'id' Value
       | 'description' STRING
       | 'mode' IDENTIFIER
       | 'listen' '[' ListenItems ']'
       | 'when' '[' ConditionItems ']'
       | 'do' '[' ActionItems ']'

ListenItems -> '(' AttrList ')' MoreListenItems | epsilon
MoreListenItems -> ',' '(' AttrList ')' MoreListenItems | epsilon

ConditionItems -> '(' AttrList ')' MoreConditionItems | epsilon
MoreConditionItems -> ',' '(' AttrList ')' MoreConditionItems | epsilon

ActionItems -> '(' ActionBody ')' MoreActionItems | epsilon
MoreActionItems -> ',' '(' ActionBody ')' MoreActionItems | epsilon

ActionBody -> IfAction | SimpleAction
IfAction -> 'if' IfCond ThenBlock ElsePart
IfCond -> AttrList | '[' ConditionItems ']'
ThenBlock -> ActionSequence
ElsePart -> 'else' ActionSequence | 'else' IfAction | epsilon

SimpleAction -> IDENTIFIER ActionRest
ActionRest -> AttrList | AttrList ',' ActionRest | epsilon

AttrList -> Attr AttrList | epsilon
Attr -> IDENTIFIER AttrValue
AttrValue -> STRING | IDENTIFIER | NUMBER | TIME | DURATION | '[' IDList ']' | epsilon

IDList -> IDENTIFIER ',' IDList | IDENTIFIER

Value -> 'auto' | NUMBER | STRING

Terminals: STRING (quoted), IDENTIFIER (atoms), NUMBER, TIME, DURATION, punctuation

This BNF describes the structure used in the mock examples: automations with fields
`id`, `description`, `mode`, `listen` (triggers), `when` (conditions) and `do` (actions),
including nested `if`/`else` actions, lists, times and durations.
